> 本文档供运维/后端工程师在阿里云 ECS 上完成首次部署与日常更新。  
> 请按章节顺序执行，每阶段完成后再进入下一阶段。

## 架构说明

```
用户浏览器
    ↓
channel.ghammerwinery.com (443 HTTPS)
    ↓
Nginx（反向代理 + /uploads 静态文件）
    ↓
Next.js :3000（PM2 进程守护）
    ↓
data/store.json + public/uploads/
```

## 阶段 0：上线前准备（阿里云控制台）

### 0.1 ECS 要求

- [ ] 已购买 ECS（建议 **2 核 2G 及以上**）
- [ ] 操作系统：**Alibaba Cloud Linux 3**（或 CentOS 7/8 系）
- [ ] 记录 ECS **公网 IP**
- [ ] 可通过 SSH 登录：`ssh root@公网IP`

### 0.2 域名 DNS 解析

在阿里云域名控制台，为 `ghammerwinery.com` 添加：

| 记录类型 | 主机记录 | 记录值 |
|----------|----------|--------|
| A | `channel` | ECS 公网 IP |

验证解析（本地或服务器均可）：

```bash
ping channel.ghammerwinery.com
```

### 0.3 安全组

ECS 安全组 **入方向** 放行：

| 端口 | 协议 | 用途 |
|------|------|------|
| 22 | TCP | SSH |
| 80 | TCP | HTTP（证书申请 + 跳转 HTTPS） |
| 443 | TCP | HTTPS |

### 0.4 ICP 备案

- [ ] ECS 位于**中国大陆**时，域名须完成 **ICP 备案** 后方可正常访问
- [ ] 香港/海外 ECS 节点可免备案

### 0.5 代码仓库

代码已托管于 GitHub，分支 **`main`**。

```bash
# 本地确认远程地址
git remote -v
# origin  git@github.com:ranmannic/ghammer.git
```

## 阶段 3：应用配置与首次启动

### 3.1 生产环境变量

```bash
cd /var/www/ghammerwinery
cp .env.example .env.production
openssl rand -base64 32
```

编辑 `.env.production`（`nano` 或 `vi`）：

```env
AUTH_SECRET=将上面 openssl 命令输出的随机字符串粘贴于此
NODE_ENV=production
PORT=3000
```

> **注意：** 切勿将 `.env.production` 提交到 Git（已在 `.gitignore` 中排除）。

### 3.2 初始化数据 + 构建

```bash
cd /var/www/ghammerwinery

npm run seed              # 首次部署：生成 data/store.json 示例数据

npm ci --omit=dev
npm run build             # 约 1~3 分钟；2G 内存不足见文末「附录 A」
```

### 3.3 PM2 启动与开机自启

```bash
pm2 start ecosystem.config.cjs
pm2 save
pm2 startup
```

执行 `pm2 startup` 后，**复制其输出的 sudo 命令再执行一次**，以设置开机自启。

验证应用：

```bash
pm2 status
# ghammerwinery 状态应为 online

curl -I http://127.0.0.1:3000
# 预期：HTTP/1.1 200
```

**阶段 3 完成标志：** 本机 `127.0.0.1:3000` 返回 200。

## 阶段 4：Nginx + 域名映射 + HTTPS

### 4.1 HTTP 临时配置（用于首次访问与申请证书）

```bash
sudo tee /etc/nginx/conf.d/channel.ghammerwinery.com.conf <<'EOF'
server {
    listen 80;
    server_name channel.ghammerwinery.com;

    location /.well-known/acme-challenge/ {
        root /var/www/certbot;
    }

    location /uploads/ {
        alias /var/www/ghammerwinery/public/uploads/;
    }

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF

sudo nginx -t && sudo systemctl reload nginx
```

- [ ] 浏览器访问 `http://channel.ghammerwinery.com`，应能看到官网首页

### 4.2 申请 HTTPS 证书（Let's Encrypt）

```bash
sudo dnf install -y certbot python3-certbot-nginx
sudo certbot --nginx -d channel.ghammerwinery.com
```

按提示：输入邮箱 → 同意条款 → 选择 **Redirect（强制 HTTPS）**。

验证证书自动续期：

```bash
sudo certbot renew --dry-run
```

**备选方案：** 使用阿里云控制台申请的免费 SSL 证书，下载 Nginx 格式后替换配置中的 `ssl_certificate` / `ssl_certificate_key` 路径。

### 4.3 应用完整 Nginx 配置

证书就绪后：

```bash
sudo cp /var/www/ghammerwinery/deploy/nginx/channel.ghammerwinery.com.conf \
  /etc/nginx/conf.d/channel.ghammerwinery.com.conf

sudo nginx -t && sudo systemctl reload nginx
```

> 若 `nginx -t` 报 ssl 证书路径错误，保留 certbot 已写入的证书路径，或手动修正 `ssl_certificate` 两行。

**阶段 4 完成标志：** `https://channel.ghammerwinery.com` 可访问，浏览器地址栏显示 HTTPS 锁标志。

## 阶段 6：日常运维

### 6.1 代码更新（Git 部署）

代码合并至 `main` 并 push 后，在 ECS 执行：

```bash
cd /var/www/ghammerwinery
./deploy/deploy.sh
```

脚本流程：`git pull` → `npm ci --omit=dev` → `npm run build` → `pm2 restart ghammerwinery`

### 6.2 数据备份（建议每周执行）

```bash
sudo tar -czf /backup/ghammer-$(date +%F).tar.gz \
  /var/www/ghammerwinery/data \
  /var/www/ghammerwinery/public/uploads
```

建议定期同步至阿里云 OSS 做异地备份。

**需备份的路径：**

- `/var/www/ghammerwinery/data/store.json` — 产品、系列、用户数据
- `/var/www/ghammerwinery/public/uploads/` — 后台上传的产品图片

### 6.3 常用运维命令

```bash
# 查看应用状态
pm2 status
pm2 logs ghammerwinery

# 重启应用
pm2 restart ghammerwinery

# 重载 Nginx
sudo nginx -t && sudo systemctl reload nginx

# 查看 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log
```

## 附录 A：构建内存不足（2G ECS）

若 `npm run build` 因内存不足失败，可临时启用 swap：

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

然后重新执行 `npm run build`。

## 附录 B：常见问题

### 502 Bad Gateway

```bash
pm2 status
curl http://127.0.0.1:3000
pm2 logs ghammerwinery --lines 50
sudo tail -20 /var/log/nginx/error.log
```

常见原因：PM2 进程未启动、3000 端口未监听、防火墙拦截。

### 上传图片失败

```bash
ls -la /var/www/ghammerwinery/public/uploads
chmod 755 /var/www/ghammerwinery/public/uploads
```

确认运行用户对 `public/uploads/` 有写权限。

### 域名无法访问

1. 检查 DNS：`ping channel.ghammerwinery.com` 是否指向 ECS IP  
2. 检查安全组：80、443 是否放行  
3. 大陆 ECS：确认 ICP 备案已完成  
4. 检查 Nginx：`sudo nginx -t` 与 `sudo systemctl status nginx`

### Git pull 失败

确认 ECS 的 SSH 公钥仍有效且已添加到 GitHub；或改用 HTTPS + Personal Access Token。
